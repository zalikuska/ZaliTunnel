# ZaliTunnel — обратный TCP туннель

Позволяет принимать входящие соединения на VPS и проксировать их на твой ПК за NAT.

```
Клиент  ──►  VPS:8080  ──►  Твой ПК:локальный_порт
Клиент  ◄──  VPS:8080  ◄──  Твой ПК (ответ)
```

## Сборка

```bash
cargo build --release
# бинарник: target/release/zalitunnel
```

## Использование

### На VPS

```bash
./zalitunnel server \
  --tunnel-port 7000 \   # сюда коннектится твой клиент
  --public-port 8080 \   # сюда коннектятся пользователи
  --pool-size 10         # сколько слотов держать наготове
```

Открой порты в файрволле:
```bash
ufw allow 7000   # для твоего клиента
ufw allow 8080   # для публичного трафика
```

### На твоём ПК

```bash
./zalitunnel client \
  --server 1.2.3.4 \          # IP твоего VPS
  --tunnel-port 7000 \
  --local 127.0.0.1:3000 \    # куда форвардить трафик локально
  --pool-size 10
```

### Пример для Minecraft

```bash
# VPS:
./zalitunnel server --tunnel-port 7000 --public-port 25565

# Твой ПК:
./zalitunnel client --server 1.2.3.4 --tunnel-port 7000 --local 127.0.0.1:25565
```

Игроки подключаются просто по `1.2.3.4` без порта.

## Протокол

Однобайтовый хэндшейк:
- `0x01` READY   — клиент говорит "я готов принять соединение"
- `0x02` RELAY   — сервер говорит "пришёл юзер, начинай релей"
- `0x03` PING    — keepalive от сервера
- `0x04` PONG    — ответ клиента

После хэндшейка — чистый TCP relay (bidirectional copy).
